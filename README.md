# 🛡️ Mkcert 增强版证书自动化工具

这是一个基于 Python + Rich 构建的 `mkcert` 图形化/交互式管理脚本。它不仅支持标准 `mkcert` 的功能，还针对 **修改版 mkcert** 进行了深度适配，支持自定义 CA/证书的组织名称、有效期等高级参数。

## 🔗 相关项目地址

- **修改版 mkcert (推荐)**: [Gitee - mr_cool/mkcert](https://gitee.com/mr_cool/mkcert/tree/master) (支持自定义元数据和超长有效期)
- **原版 mkcert**: [GitHub - FiloSottile/mkcert](https://github.com/FiloSottile/mkcert)

## ✨ 功能特性

- 🔑 **根证书 (CA) 管理**：
  - 支持交互式申请自定义 CA（可自定义组织名称、部门、通用名称及有效期年限等）。
  - 自动检测系统 CA 状态，显示物理路径及确切过期时间。
  - **安全机制**：在覆盖现有 CA 前会强制弹出风险提示，防止旧证书失效。
- 🆕 **证书申请**：
  - 支持多域名、通配符域名及 IP 申请。
  - 支持交互式自定义证书元数据（组织名称、组织部门、通用名称及有效期天数等）。
  - 自动忽略同一次输入中的重复域名；支持自定义证书文件名，覆盖同名证书前会二次确认。
- 📜 **证书一览**：
  - 自动扫描并展示已生成的证书，并按剩余天数**升序排列**（临期证书优先展示）。
  - **智能解析**：自动提取证书内的域名信息（SAN）及过期时间。
- ⚡ **快速续期**：
  - 在一览列表中通过序号即可“一键续期”，按 `config.ini` 中的 `renew_days` 重新生成证书。
- 🗑️ **安全删除**：
  - 在证书一览中输入 `del 序号` 可删除对应证书和私钥，删除前会二次确认。
- 🧪 **配置校验**：
  - 启动时校验有效期、预警阈值、证书目录、默认域名和 `mkcert` 路径，配置错误会直接提示。
- 🧾 **操作日志**：
  - 自动记录启动、申请、续期、删除和 CA 操作，默认写入 `logs/mkcert.log`。
- 📦 **环境管理**：基于 `uv` 构建，依赖隔离，运行速度极快。

## 🚀 快速开始

### 1. 前置要求

- 已安装 [uv](https://github.com/astral-sh/uv) (强大的 Python 包管理器)。
- 已将 **`mkcert.exe`** (修改版或标准版) 放置在项目根目录、`bin/` 目录，或在 `config.ini` 中配置 `mkcert_path`。

### 2. 初始化环境

在项目根目录下运行：

```powershell
uv sync
```

### 3. 运行工具

```powershell
uv run main.py
```

## 🛠️ 功能指南

### 主菜单说明

- **1. 证书一览**：查看所有已生成证书。输入序号续期，输入 `del 序号` 删除对应证书和私钥，输入 `0` 返回。
- **2. 申请证书**：输入域名后按提示操作，可确认或自定义证书文件名。若需自定义有效期，请在相应选项中输入天数（如 `3650`）；如目标证书文件已存在，会提示确认是否覆盖。
- **R. 申请 CA**：安装或更换根证书。支持设置 `ca-years`（如 15 年）。
- **Q. 退出脚本**：安全退出程序。

### 有效期说明

- **公网浏览器信任证书**：公开 CA 签发的 TLS/SSL 证书受 CA/Browser Forum 和浏览器根计划限制；当前最长有效期已不再是 825 天，实际限制会随行业规则继续缩短。
- **本地自签证书**：`mkcert` 使用本地私有 CA 签发证书，手动信任该 CA 后通常不受公网证书最长有效期规则直接限制。
- **推荐设置**：个人本地开发可按需设置较长有效期，例如 `3650` 天；团队或跨设备共享时建议缩短有效期，并建立定期续期习惯。

## ⚠️ 注意事项

- **根证书覆盖风险**：重新申请根证书 (CA) 会导致之前签发的所有域名证书失效。除非必要，请勿随意重新申请 CA。
- **CA 到期预警**：主界面会按 `warn_days_yellow` / `warn_days_red` 对根证书剩余天数进行预警；CA 过期后，已签发证书通常也会失去信任链。
- **私钥安全**：`certs/` 目录中的 `*-key.pem` 是域名证书私钥，请勿提交到 Git、网盘公开目录或聊天工具；如果怀疑私钥泄露，应立即重新签发证书。
- **覆盖证书风险**：重新申请同名证书会覆盖原证书和私钥，正在使用该证书的本地服务通常需要重载后才会使用新证书。
- **删除证书风险**：删除域名证书不会影响根 CA，但会影响正在使用该证书的本地服务；删除后如需继续使用，只能重新签发。
- **CA 私钥风险**：`mkcert` 的根 CA 私钥可用于签发被本机信任的任意本地域名证书，泄露风险高于普通域名私钥；不要把 CA 根目录整体分享给他人。
- **共享证书原则**：跨设备使用时只分发需要信任的 `rootCA.pem` 或具体证书文件，不要分发 `rootCA-key.pem`、`*-key.pem` 等私钥文件。
- **日志文件**：日志默认位于 `logs/mkcert.log`，可能包含证书文件路径和域名信息；如果共享项目目录，请按需清理或避免提交日志。
- **编码支持**：脚本已针对 Windows 控制台做了兼容性处理，支持在 CMD 或 PowerShell 中完美显示中文和 Emoji。
